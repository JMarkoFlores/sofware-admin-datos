🚀 FLUJOS PRINCIPALES

FLUJO 1: ACTIVACIÓN
  1. Usuario abre panel Disparador → sección Reportes Automáticos
  2. Ingresa número de WhatsApp
  3. Hace clic en "Iniciar Reportes"
  4. Backend programa 2 APScheduler jobs:
     - Diario: 08:00 AM America/Lima (configurable)
     - Semanal: Lunes 09:00 AM (configurable)

FLUJO 2: REPORTES AUTOMÁTICOS DIARIOS
  1. Se ejecuta CronTrigger diario
  2. Para cada tipo en REPORTS_DAILY_TYPES:
     - generar_reporte(tipo) → genera contenido
     - send_message(contenido) → envía por WhatsApp
  3. Registra en auditoría

FLUJO 3: REPORTES AUTOMÁTICOS SEMANALES
  1. Se ejecuta CronTrigger semanal (Lunes)
  2. Para cada tipo en REPORTS_WEEKLY_TYPES:
     - generar_reporte(tipo) → genera contenido
     - send_message(contenido) → envía por WhatsApp
  3. Registra en auditoría

FLUJO 4: REPORTES INTERACTIVOS BAJO DEMANDA
  1. Usuario envía mensaje a WhatsApp
  2. Evolution API POST a /api/backup/webhook
  3. Sistema identifica usuario por teléfono
  4. procesar_mensaje_usuario() gestiona estados:
     idle → muestra menú
     menu → valida selección
     generating_report → genera contenido
     report_ready → ofrece continuar
  5. UserConversation persiste estado en DB
  6. Siguiente mensaje continúa desde el estado anterior

⚙️ CONFIGURACIÓN

Archivo: dashboard/backend/.env

```env
# Reportes Automáticos - DIARIOS
REPORTS_DAILY_HOUR=8              # Hora del día (0-23)
REPORTS_DAILY_MINUTE=0            # Minuto (0-59)
REPORTS_DAILY_TYPES=estadisticas,multas  # Tipos separados por coma

# Reportes Automáticos - SEMANALES
REPORTS_WEEKLY_DAY=MON            # MON|TUE|WED|THU|FRI|SAT|SUN
REPORTS_WEEKLY_HOUR=9             # Hora (0-23)
REPORTS_WEEKLY_MINUTE=0           # Minuto (0-59)
REPORTS_WEEKLY_TYPES=estadisticas,libros_prestados,devoluciones_pendientes

# Reportes - TIMEZONE
REPORTS_TIMEZONE=America/Lima     # Zona horaria para CronTrigger
```
